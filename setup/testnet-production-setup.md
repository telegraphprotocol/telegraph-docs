# Testnet & Production Setup

This guide explains how to set up a Telegraph validator node for production or testnet: **network config**, **subnets.yaml**, running **telegraph.sh**, and using the **test scripts**.

## Overview

1. **Get config from Telegraph Devs** – `networks.config.json` (Diamond addresses and RPCs per chain) and **subnets.yaml** (subnet API endpoints and auth). Place them in the repo root.
2. **Deploy contracts (if needed)** – Run `./deploy-testnet.sh` to deploy Diamonds to testnets (e.g. Sepolia, Fuji) and update `networks.config.json`. Skip if you already have config from the team.
3. **Run telegraph.sh** – The script installs dependencies, sets up Cassandra, loads networks from `networks.config.json`, configures Clef and Telegraph, and prompts for **this node’s IP** and **Genesis node IP** (the latter is typically shared by Telegraph Devs).
4. **Fund validator** – Fund the Clef wallet on each chain (script can do this optionally with a deployer key).
5. **Verify** – Use `./tests/check-registration.sh`, `./tests/bridge-test.sh`, and `./tests/subnet-inference-test.sh` (choose Testnet and the right networks).

**Important:** `telegraph.sh` does **not** deploy contracts. Use `deploy-testnet.sh` first for testnets, or get a pre-filled `networks.config.json` and `subnets.yaml` from Telegraph Devs.

## Prerequisites

- **Linux/Unix system** (tested on Ubuntu/Debian)
- **Go 1.19+** (installed by script if missing)
- **Java 11+** (installed by script if missing)
- **Cassandra** (installed by script if missing)
- **Root/sudo access** (for systemd services)
- **networks.config.json** and **subnets.yaml** in the repo root (from Telegraph Devs or from your own deploy)
- **Genesis node IP** (shared by Telegraph Devs for your environment)

## 1. Network config: networks.config.json

**Source:** Shared by Telegraph Devs or generated by `./deploy-testnet.sh`.

Place `networks.config.json` in the **project root**. It defines chains and Diamond (Port) addresses. The node and test scripts read it (and `telegraph.sh` syncs it into Cassandra).

**Structure:**

```json
{
  "testnet": [
    {
      "chainId": 11155111,
      "name": "Sepolia-ETH",
      "type": "EVM",
      "rpcHttp": "https://ethereum-sepolia-rpc.publicnode.com",
      "rpcWs": "wss://ethereum-sepolia-rpc.publicnode.com",
      "diamondAddress": "0x..."
    },
    {
      "chainId": 43113,
      "name": "Avalanche-Fuji",
      "type": "EVM",
      "rpcHttp": "https://avalanche-fuji-c-chain-rpc.publicnode.com",
      "rpcWs": "wss://avalanche-fuji-c-chain-rpc.publicnode.com",
      "diamondAddress": "0x..."
    }
  ],
  "mainnet": [
    { "chainId": 1, "name": "Ethereum", "type": "EVM", "rpcHttp": "...", "rpcWs": "...", "diamondAddress": "0x..." }
  ]
}
```

- **chainId** – EVM chain ID.
- **name** – Unique chain name (e.g. `"Sepolia-ETH"`, `"Avalanche-Fuji"`). Used for cross-chain routing; must match what Telegraph Devs use.
- **rpcHttp** / **rpcWs** – RPC URLs for that chain.
- **diamondAddress** – Deployed Telegraph Diamond (Port) contract. Required; the script skips entries with empty `diamondAddress`.

If you deploy yourself, run `./deploy-testnet.sh` (see below); it updates the `testnet` section with deployed addresses.

## 2. Subnet config: subnets.yaml

**Source:** Shared by Telegraph Devs.

Place `subnets.yaml` in the **project root**. The node uses it for subnet API endpoints, auth, and rate limits. `telegraph.sh` sets `SUBNET_DISPATCHER_CONFIG` to this path in `~/.env`.

**Example (excerpt):**

```yaml
global:
  server_port: 8080
  default_timeout_seconds: 15
  log_level: info

subnets:
  - name: 34   # e.g. Bitmind
    base_url: "https://api.bitmind.ai/oracle/v1"
    auth:
      type: header
      env_key: bitmind-...
    endpoints:
      - path: /34/detect-image
        externalPath: /detect-image
        method: POST
  # ... more subnets
```

Do not remove or rename subnets the network relies on. Add API keys in `telegraph.sh` or in `~/.env` (e.g. `SUBNET_34_API_KEY`) as required by the auth config in `subnets.yaml`.

## 3. Deploying contracts (testnet)

If you are not using a pre-filled config from Telegraph Devs, deploy Diamonds to testnets and update `networks.config.json`:

```bash
chmod +x deploy-testnet.sh
./deploy-testnet.sh
```

You will be prompted for:
- **Deployer private key** (hex)
- **Which networks** to deploy to (default: Sepolia and Fuji)

The script uses Foundry (`forge script`) and updates the **testnet** section of `networks.config.json` with the new Diamond addresses. Then run `telegraph.sh` so the DB and node use the updated config.

## 4. Running telegraph.sh and config

Run the setup script from the project root:

```bash
chmod +x telegraph.sh
./telegraph.sh
```

The script:

1. **Dependencies** – Installs Go, Java, Cassandra, jq, expect if needed.
2. **Networks from config** – Reads `networks.config.json`; only entries with a valid `diamondAddress` are loaded. Exits if none.
3. **Cassandra** – Applies `schema.cql`, runs `migrations/*.cql`, truncates `telegraph.network`, and inserts the loaded networks.
4. **Subnet config in DB** – Inserts default subnet rows if not present (script may also use `subnets.yaml`-derived data where applicable).
5. **Clef** – Downloads/configures Clef, creates systemd service, generates wallet.
6. **Prompts** – Asks for:
   - **This node’s IP address** (default `127.0.0.1`) and **port** (default `7044`). Use the **public IP/hostname** other nodes and the Genesis node will use to reach this node.
   - **Genesis node IP** (and port). **This is typically shared by Telegraph Devs** (e.g. `http://genesis.example.com:7044`). For a single-node test you can use this node’s own URL.
   - **Node moniker** (name), e.g. `TestnetNode`.
7. **Validator entry** – Creates/updates `telegraph.validator` with Clef address, moniker, and domain (this node’s URL).
8. **Optional funding** – Asks whether to fund the validator wallet on each testnet chain using a deployer private key (for gas). You can skip and fund manually.
9. **Build & .env** – Builds Telegraph, writes `~/.env` with `CHAIN_ID`, `CONTRACT_ADDRESS`, `IP`, `GENESIS_IP`, `MONIKER`, `EXTERNAL_SIGNER_URL`, `SUBNET_DISPATCHER_CONFIG=<path/to/subnets.yaml>`, `AUTO_REGISTER_ALL_NETWORKS=true`, and subnet API keys.
10. **Systemd** – Creates and starts `telegraph.service`.

After this, the node loads all networks from the DB and, with `AUTO_REGISTER_ALL_NETWORKS=true`, registers and listens on every Diamond automatically.

## 5. Example test scripts

From the **project root**, after the node is running:

### check-registration.sh

Checks node status, Clef, wallet balance on configured networks, and validator DB entry.

```bash
./tests/check-registration.sh
```

- Choose **(2) Testnet** when prompted.
- Optionally choose **Network A** / **Network B** (default Sepolia / Fuji). The script uses `networks.config.json` testnet section and reports balances for the Clef wallet on those chains.

### bridge-test.sh

Cross-chain message test (e.g. Sepolia → Fuji).

```bash
./tests/bridge-test.sh
```

- Choose **(2) Testnet**, then **Network A** (origin, e.g. Sepolia) and **Network B** (destination, e.g. Fuji).
- Choose **(d)** to deploy a new `BridgeReceiverTestApp` on destination or **(e)** to use an existing address.
- Optionally set **Bridge message** (default: `Hello from bridge test`). The script deposits gas on destination, emits an outbound message from origin, waits for relay (10s / 20s / 30s), and verifies the message on the receiver contract.

### subnet-inference-test.sh

Same-chain subnet inference test (e.g. Bitmind on Sepolia).

```bash
./tests/subnet-inference-test.sh
```

- Choose **(2) Testnet** and **Network** (e.g. Sepolia).
- Script deploys `SubnetTestApp`, deposits gas, sends a subnet request, and verifies the response on the callback contract after a short wait.

**Requirements:** `forge`, `cast`, and `jq`. For testnet, the script uses the testnet private key (or set `PRIVATE_KEY`). Ensure the validator wallet has gas on the relevant chains and that `networks.config.json` testnet section is correct.

## What the Script Does (detail)

### 1. Database Setup

- Creates Cassandra keyspace `telegraph` if it doesn't exist
- Applies schema from `schema.cql`
- Runs migrations from `migrations/` directory
- **Truncates** `telegraph.network` and inserts one row per chain from `networks.config.json` (testnet + mainnet) that has a valid `diamondAddress`
- Inserts subnet configuration rows if missing
- Uses **local cqlsh** (no Docker)

### 2. Clef Configuration

- Downloads Clef binary if not present
- Creates systemd service `clef.service`
- Generates wallet account automatically
- Configures expect script for password automation
- Starts Clef service

### 3. Network Configuration

- Reads `networks.config.json` from project root
- Parses `testnet` and `mainnet` sections; only entries with a valid hex `diamondAddress` are loaded
- Truncates `telegraph.network` and inserts one row per loaded network so the DB matches the file

### 4. Validator Entry

- Extracts Clef wallet address from logs
- Creates/updates validator entry in `telegraph.validator`:
  - `publicethaddress`: Clef account address
  - `moniker`: From `MONIKER` env var
  - `domain`: From `IP` env var (must be reachable by other nodes)
  - `ismine`: `true`

### 5. Telegraph Node Configuration

- Builds Telegraph from source (if `main.go` present in repo)
- Creates `.env` file in `$HOME/.env` with:
  - `CHAIN_ID`, `CONTRACT_ADDRESS`, `EVM_HTTP_URL`, `EVM_WSS_URL` from the first network in config
  - `IP`: this node’s URL (from prompt)
  - `GENESIS_IP`: Genesis node URL (from prompt; **get this from Telegraph Devs**)
  - `MONIKER`, `EXTERNAL_SIGNER_URL`, `AUTO_REGISTER_ALL_NETWORKS=true`
  - `SUBNET_DISPATCHER_CONFIG`: path to repo `subnets.yaml`
  - Subnet API keys (SUBNET_18_API_KEY, etc.)
- Creates systemd service `telegraph.service` and starts it

### 6. Auto-Registration

- Node automatically registers as signer on all configured networks
- Triggered on startup (after 10 second delay)
- Uses `AUTO_REGISTER_ALL_NETWORKS=true` flag
- Updates `chain_id.json` before each registration

## Verification Steps

### 1. Check Clef Service

```bash
sudo systemctl status clef.service
sudo journalctl -u clef.service -f
```

### 2. Check Telegraph Service

```bash
sudo systemctl status telegraph.service
sudo journalctl -u telegraph.service -f
```

### 3. Verify Database Networks

```bash
cqlsh -e "SELECT chainid, name, contractaddress FROM telegraph.network;"
```

Expected output should show all testnet and mainnet networks with their Diamond addresses.

### 4. Verify Validator Entry

```bash
cqlsh -e "SELECT publicethaddress, moniker, domain, ismine FROM telegraph.validator WHERE ismine = true;"
```

### 5. Verify Node is Registered as Signer

Check Telegraph logs for registration confirmations:

```
Successfully registered as signer on Sepolia-ETH (ChainID: 11155111)
Successfully registered as signer on BNB (ChainID: 56)
...
```

Or verify on-chain using Forge/cast:

```bash
cd contracts
forge script script/CheckSigner.s.sol --rpc-url <RPC_URL>
```

### 6. Check Auto-Registration Status

Look for these log messages:

```
AUTO_REGISTER_ALL_NETWORKS is enabled, ensuring node is registered on all networks...
Starting auto-registration on all networks (N networks)
Successfully registered as signer on <Network Name> (ChainID: <ID>)
Auto-registration on all networks completed
```

### 7. Run example test scripts

Use `./tests/check-registration.sh`, `./tests/bridge-test.sh`, and `./tests/subnet-inference-test.sh` (choose Testnet). See **§5. Example test scripts** above.

### 8. Test RPC connectivity

```bash
curl -X POST -H "Content-Type: application/json" \
  --data '{"jsonrpc":"2.0","method":"eth_blockNumber","params":[],"id":1}' \
  <RPC_URL>
```

## Troubleshooting

### Networks Not Parsed from Config

**Problem:** Script fails to parse `networks.config.json`

**Solutions:**
- Install `jq` for reliable JSON parsing: `sudo apt-get install jq`
- Verify JSON syntax: `cat networks.config.json | jq .`
- Check file exists: `ls -l networks.config.json`
- Ensure all networks have non-empty `diamondAddress`

### Contract Addresses Missing

**Problem:** Networks skipped due to empty contract addresses

**Solutions:**
- Verify `networks.config.json` has `diamondAddress` for all networks
- Check Diamond contracts are deployed on all networks
- Update config file with correct addresses

### Cassandra Connection Issues

**Problem:** Cannot connect to Cassandra

**Solutions:**
- Check Cassandra is running: `cassandra -v`
- Wait longer for Cassandra to start (can take 60+ seconds)
- Check Cassandra logs: `tail -f /var/log/cassandra/system.log`
- Verify port 9042 is open: `netstat -tlnp | grep 9042`
- Check firewall rules if Cassandra is remote

### Clef Service Not Starting

**Problem:** Clef service fails to start

**Solutions:**
- Check Clef logs: `sudo journalctl -u clef.service -f`
- Verify Clef binary exists: `ls $BRIDGE_DIR/clefBin/clef`
- Check expect script permissions: `ls -l $BRIDGE_DIR/scripts/clef_service.exp`
- Verify password is set correctly in script
- Check disk space: `df -h`

### Node Not Auto-Registering

**Problem:** Node doesn't register as signer automatically

**Solutions:**
- Check `.env` has `AUTO_REGISTER_ALL_NETWORKS=true`
- Verify `EXTERNAL_SIGNER_URL` is set correctly
- Check Clef is running and accessible
- Look for errors in Telegraph logs
- **Ensure wallet has funds on all networks** (for gas fees)
- Verify RPC URLs are accessible from server
- Check network connectivity: `ping <RPC_HOST>`

### Registration Transaction Fails

**Problem:** Registration transactions fail on some networks

**Solutions:**
- Check wallet balance on that network
- Verify RPC URL is correct and accessible
- Check gas prices aren't too high
- Verify Diamond contract address is correct
- Check network is not paused/maintenance mode
- Review transaction receipt in logs

### Migration Failures

**Problem:** Database migrations fail

**Solutions:**
- Check migration files exist: `ls migrations/*.cql`
- Verify Cassandra is accessible: `cqlsh -e "DESCRIBE KEYSPACES;"`
- Check migration script: `cat scripts/migrate.sh`
- Try running migrations manually: `make db-migrate`
- Check for schema conflicts with existing data

## Manual Steps

### Fund the Clef Wallet

**Critical:** Fund the Clef wallet on ALL networks before running the node:

```bash
# Get wallet address from Clef logs
WALLET=$(grep -oE "0x[a-fA-F0-9]{40}" $BRIDGE_DIR/clef_script.log | head -1)

# Fund on each network (example for Sepolia)
cast send $WALLET --value 0.1ether \
  --rpc-url https://ethereum-sepolia-rpc.publicnode.com \
  --private-key <YOUR_FUNDING_KEY>

# Repeat for all networks in networks.config.json
```

**Minimum funding recommendations:**
- Testnets: 0.1 ETH equivalent per network
- Mainnets: 0.5 ETH equivalent per network
- Consider gas price fluctuations

### Backup Keystore

**Important:** Backup the Clef keystore immediately:

```bash
# Backup keystore directory
tar -czf clef-keystore-backup-$(date +%Y%m%d).tar.gz $BRIDGE_DIR/clef/keystore/

# Store backup securely (offline, encrypted)
```

### Update network config

If Diamond addresses or RPCs change, update `networks.config.json` (or get an updated file from Telegraph Devs), then re-run:

```bash
./telegraph.sh
```

The script reloads networks and repopulates the DB; restart the service if it’s already running.

### Manually Trigger Registration

If auto-registration fails, manually trigger:

```bash
# Restart Telegraph service (will trigger auto-registration)
sudo systemctl restart telegraph.service

# Or wait for next startup cycle
```

## Security Considerations

### Firewall Configuration

Ensure these ports are accessible:
- **7044**: Telegraph HTTP API (for other nodes)
- **9042**: Cassandra (restrict to localhost or VPN)
- **Clef IPC**: Local filesystem only

### Password Security

- Use strong `CLEF_PASSWORD` (20+ characters recommended)
- Store passwords securely (use password manager)
- Rotate passwords periodically
- Never commit passwords to git

### Keystore Security

- Backup keystore immediately after generation
- Store backups encrypted and offline
- Limit filesystem permissions: `chmod 600 $BRIDGE_DIR/clef/keystore/*`
- Consider hardware wallet integration for production

### Network Security

- Use HTTPS/WSS for RPC URLs when possible
- Verify RPC provider authenticity
- Monitor for suspicious activity
- Keep node software updated

## Maintenance

### Updating Networks

When new networks are added or addresses change:

1. Update `networks.config.json`
2. Re-run `telegraph.sh` (will update existing entries)
3. Restart Telegraph service: `sudo systemctl restart telegraph.service`

### Updating Node Software

1. Stop services: `sudo systemctl stop telegraph.service`
2. Backup database and keystore
3. Update Telegraph binary
4. Restart services: `sudo systemctl start telegraph.service`

### Database Backups

```bash
# Backup keyspace
nodetool snapshot telegraph

# Restore from backup
# (See Cassandra documentation for restore procedures)
```
