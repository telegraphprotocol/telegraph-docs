# Time Based Thresholds

As a community driven bridge, Telegraph cannot rely on a central entity to determine which transactions are authorized to move through the network. To solve this issue, the network uses time as factor in determining the weight of each participating node's signature. This effectively places strong emphasis on participating within the network early, as nodes that have more time within the network will have a larger voice.

When a network like Telegraph's grows, the threat of sabotage and theft from malicious actors grows as well--and under evenly distributed circumstances, unscrupulous actors could attempts to **51% attack** the network by paying the validator fees for a validator botnet they control. Utilizing time as a weight effectively renders the attack useless and gives honest network participants a window to vote out malicious actors.
